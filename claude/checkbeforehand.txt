```
### 2. **プロジェクトの要件確認**

#### ✅ 必須確認項目
- [ ] **データ量の想定**: 調査地は何箇所くらいを想定？（100箇所？1000箇所？）
- [ ] **既存データの有無**: すでにExcel/CSVでデータを持っている？→インポート機能を優先実装
- [ ] **優先機能**: 最初に使いたい機能は？
  - データ入力機能
  - 既存データのインポート
  - 行列変換・CSV出力
  - 地図表示
- [ ] **植生データの項目**: 要件定義書の項目で足りる？追加・削除したい項目は？
- [ ] **アリ種名の管理**: すでに種名リストがある？→初期データとして投入

#### 📋 オプション確認
- インターネット接続環境: 常時接続？オフライン環境もある？
- 画面サイズ: 使用するPCの解像度は？（推奨: 1920×1080以上）
- 複数PCでの使用: 1台だけ？複数台で使う可能性は？

### 3. **開発方針の選択**

以下のどちらで進めますか？

#### 🅰️ **段階的開発（推奨）**
```
Phase 1: データベース + 基本CRUD（2週間）
↓ ← ここで動作確認・フィードバック
Phase 2: データ入力UI（2週間）
↓ ← ここで実際にデータ入力してみる
Phase 3: 解析・出力機能（2週間）
...
```
**メリット**: 早い段階で動くものを確認でき、方向修正しやすい

#### 🅱️ **一括開発**
```
全機能を実装してから動作確認
```
**メリット**: 最初から完成形を目指せる  
**デメリット**: 問題が見つかった時の修正コストが大きい

## 📁 プロジェクト構造（提案）

以下の構造で作成します：
```
ant_research_system/
├── main.py                    # アプリケーションのエントリーポイント
├── requirements.txt           # 依存パッケージリスト
├── config.ini                 # 設定ファイル
├── README.md                  # プロジェクト説明
│
├── models/                    # データモデル層
│   ├── __init__.py
│   ├── database.py           # データベース接続・初期化
│   ├── parent_site.py        # 親調査地モデル
│   ├── survey_site.py        # 調査地モデル
│   ├── survey_event.py       # 調査イベントモデル
│   ├── vegetation.py         # 植生データモデル
│   ├── ant_record.py         # アリ類記録モデル
│   └── species.py            # 種名マスタモデル
│
├── views/                     # ビュー層（GUI）
│   ├── __init__.py
│   ├── main_window.py        # メインウィンドウ
│   ├── input_tab.py          # 入力タブ
│   ├── view_tab.py           # 閲覧タブ
│   ├── analysis_tab.py       # 解析タブ
│   ├── map_tab.py            # 地図タブ
│   └── settings_tab.py       # 設定タブ
│
├── controllers/               # コントローラ層
│   ├── __init__.py
│   ├── site_controller.py    # 調査地管理
│   ├── data_controller.py    # データ管理
│   ├── export_controller.py  # エクスポート処理
│   ├── analysis_controller.py # 解析処理
│   └── map_controller.py     # 地図処理
│
├── utils/                     # ユーティリティ
│   ├── __init__.py
│   ├── validators.py         # バリデーション
│   ├── logger.py             # ログ処理
│   ├── backup.py             # バックアップ処理
│   └── geo_utils.py          # 地理計算
│
├── data/                      # データディレクトリ
│   ├── ant_database.db       # SQLiteデータベース（自動生成）
│   └── init_schema.sql       # データベース初期化SQL
│
├── backups/                   # バックアップ保存先
│
├── logs/                      # ログファイル保存先
│
├── exports/                   # エクスポートファイル保存先
│
├── templates/                 # テンプレートファイル
│   ├── import_template_sites.csv
│   ├── import_template_vegetation.csv
│   └── import_template_ants.csv
│
└── tests/                     # テストコード
    ├── __init__.py
    ├── test_models.py
    ├── test_validators.py
    └── test_geo_utils.py
```

## 🚀 開発の進め方（提案）

### **推奨アプローチ: MVP（Minimum Viable Product）方式**

#### Step 1: 最小限の動くシステム（1-2日）
```
✓ データベース作成
✓ 親調査地の登録・一覧表示
✓ GUIの基本構造
```
→ **ここで一度動作確認！**

#### Step 2: 基本機能の拡充（3-5日）
```
✓ 調査地の登録
✓ 調査イベントの登録
✓ 植生データの入力
```
→ **実際にデータを入れてみる**

#### Step 3: アリ類データ（3-5日）
```
✓ 種名マスタ管理
✓ アリ類記録の登録
✓ 出現リスト表示
```

#### Step 4: 解析・出力（5-7日）
```
✓ 行列変換
✓ CSV出力
✓ 散布図
```

#### Step 5: 地図・高度な機能（残り）
```
✓ 地図表示
✓ クラスタ解析
✓ バックアップ
